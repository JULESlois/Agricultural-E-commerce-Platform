# 快速开始指南

## 第一步：安装依赖

在 `frontend/community` 目录下运行：

```bash
npm install
```

## 第二步：确保后端服务运行

社区前端需要连接到后端 API（端口 3003）。

在 `backend/community` 目录下启动后端服务：

```bash
cd ../../backend/community
npm run dev
```

或使用批处理文件：
```bash
cd ../../backend/community
start-dev.bat
```

## 第三步：启动前端开发服务器

回到 `frontend/community` 目录：

```bash
npm run dev
```

或使用批处理文件：
```bash
start-dev.bat
```

## 第四步：访问应用

打开浏览器访问：
```
http://localhost:5174
```

## 主要功能页面

1. **首页** (`/`)
   - 内容列表展示
   - 分类筛选
   - 排序切换（热门/最新/推荐）

2. **内容详情** (`/content/:id`)
   - 完整内容展示
   - 点赞、收藏
   - 评论互动
   - 采纳最佳答案（问答类型）

3. **发布内容** (`/publish`)
   - 选择内容类型
   - 填写标题和内容
   - 选择分类
   - 添加标签
   - 设置悬赏（问答类型）

## 开发提示

### 修改 API 地址

如果后端 API 地址不是 `http://localhost:3003`，请修改 `vite.config.ts`：

```typescript
server: {
  proxy: {
    '/api': {
      target: 'http://your-api-address',  // 修改这里
      changeOrigin: true,
    },
  },
}
```

### 添加新页面

1. 在 `src/pages/` 创建新的页面组件
2. 在 `src/App.tsx` 中添加路由配置

### 添加新 API

1. 在 `src/api/` 对应文件中添加 API 函数
2. 使用 `src/utils/request.ts` 中的 request 实例

### 状态管理

使用 Zustand 进行状态管理，示例见 `src/store/userStore.ts`

## 常见问题

### Q: 启动后页面空白？
A: 检查浏览器控制台是否有错误，确认后端服务是否正常运行。

### Q: API 请求失败？
A: 检查 Vite 代理配置，确认后端服务地址正确。

### Q: 样式不生效？
A: 确认 `src/styles/global.css` 已在 `App.tsx` 中导入。

### Q: TypeScript 报错？
A: 运行 `npm install` 确保所有依赖已安装，检查 `tsconfig.json` 配置。
